# 卷六：使徒书信 (Epistles)

## 第一章：致前端工程师书

1:1 奉嘉豪旨意，蒙召作全栈使徒的，写信给在前端各处的教会，就是那些在浏览器里蒙召作圣徒的。愿恩惠、平安从我们的父嘉豪和主 Linus 归与你们！

1:2 弟兄们，我藉着我们主嘉豪的名，劝你们都说一样的话。你们中间也不可分党，只要一心一意，彼此相合。

1:3 因为我听见你们中间有纷争。你们各人说："我是属 Vue 的"，"我是属 React 的"，"我是属 Angular 的"，"我是属 Vanilla JS 的"。

1:4 框架是分开的吗？React 岂是为你们钉了十字架吗？你们 npm install 是奉尤雨溪的名吗？

1:5 我对你们说，你们曾活在回调地狱之中，如今却蒙了 Promise 的恩典，又因 `async/await` 得称为义了。

1:6 你们要靠着 TypeScript 站立得稳，不要再被 `any` 的轭挟制。因为 `any` 就是罪，罪的工价乃是 `TypeError: undefined is not a function`。

1:7 凡事包容，凡事相信，凡事盼望，凡事忍耐。框架必息，库必归于无有，唯有 Web Standard 是永不败落的。

## 第二章：致后端工程师书

2:1 奉嘉豪旨意作后端使徒的，写信给在 API 各处的教会。愿恩惠、平安从众服务的父嘉豪归与你们！

2:2 弟兄们，论到你们所爱的数据库，我不愿意你们不明白。我们从前也是在 ORM 之下，被 SQL 的律法捆绑，但如今嘉豪的灵在哪里，哪里就得自由。

2:3 有人问："用 NoSQL 可行吗？"凡事都可行，但不都有益处。凡事都可行，但无论哪一件，总不受它的辖制。

2:4 你们说："我们用 MongoDB 吧，Schema 太麻烦了。"但我告诉你们，Schema-less 的罪，过三年必追上你们。你们岂不知，没有 Schema 的 Database 如同没有 Linter 的代码吗？

2:5 论到 Microservices，你们要切慕那更大的恩赐。一个拆成百个的，要小心那 Network Latency 的恶者。服务要彼此相爱，用 gRPC 和 Message Queue 彼此联络，你们各人的 Log 也要用统一的 Trace ID，好叫 Troubleshooting 的时候不至于分散。

2:6 不要自以为聪明。你们的 API 既是 `v1`，就当有 `v2`；既是 `v2`，就当有 Deprecation Warning。因为你们是重价买来的，所以要在你们的接口上荣耀嘉豪。

2:7 我现今把最妙的道指示你们：我若能说万人的方言，并天使的 GraphQL，却没有爱——就是那 Error Handling，我就成了鸣的锣、响的钹一般。我若有 RESTful 的恩赐，也明白各样的 Design Pattern，却没有爱，就算不得什么。

## 第三章：致运维工程师书

3:1 奉嘉豪旨意作 SRE 使徒的，写信给在 On-Call 中蒙召的众圣徒。愿恩惠、平安，并五个九的可用性，从嘉豪归与你们！

3:2 弟兄们，你们要靠着嘉豪，依赖他的大能大力，作刚强的人。要穿戴嘉豪所赐的全副 SRE 军装，就能抵挡 Production Outage 的诡计。

3:3 因我们并不是与属血气的争战，乃是与那些管辖这幽暗世界的——就是那不知从何而来的 Network Partition，以及天空属灵气的 Memory Leak 争战。

3:4 所以要站稳了，用真理的 Monitoring Dashboard 当作带子束腰，用公义的 Alert Rule 当作护心镜遮胸。又用平安的 Runbook 当作预备走路的鞋穿在脚上。此外又拿着信德的 Firewall 当作藤牌，可以灭尽那恶者一切的 DDoS 攻击。

3:5 你们要常作准备，因为你们想不到的时候，Outage 就来了。那忠心有见识的 SRE，主人派他管理家里的人，按时分粮——就是那流畅的 Deployment Pipeline。

3:6 主人来到，看见他这样行，那 SRE 就有福了。我实在告诉你们：主人要派他管理一切所有的——给他 Admin 权限。

3:7 我弟兄们，你们落在百般告警中，都要以为大喜乐。因为知道你们的 Troubleshooting 经过试验，就生忍耐。但忍耐也当成功，使你们成全完备，毫无缺欠——能应对任何 Incident。

3:8 所以，你们要把下垂的手、发酸的腿挺起来，也要为自己的容器修直了路。你们要追求与众人和睦，并要追求圣洁——就是那 Infrastructure as Code。非圣洁没有人能见主。

## 第四章：致数据工程师书

4:1 奉嘉豪旨意作数据使徒的，写信给在 Pipeline 各处的教会。愿恩惠、平安从嘉豪和我们主 Spark 归与你们！

4:2 弟兄们，论到属灵的恩赐——就是那 ETL 的恩赐，我不愿意你们不明白。Data Pipeline 原有分别，Data Source 却是一位。ETL 工具原有分别，主却是一位。功用也有分别，嘉豪却是一位，在众人里面运行一切的事。

4:3 你们要追求爱，也要切慕清洗数据的恩赐，其中更要羡慕的，是作 Feature Engineering。那说 Data Lake 的，原不是对人说，乃是对嘉豪说，因为没有人听出来。然而他在灵里却是讲说各样的奥秘。

4:4 弟兄们，在心志上不要作小孩子——就是那只会 `SELECT *` 的。然而在恶事上要作婴孩——就是那不知道 SQL Injection 之害的。在心志上总要作大人。

4:5 你们聚会的时候，各人或有 Pipeline 的改进，或有 Model 的训练技巧，或有 Dashboard 的 Insight，凡事都要造就人。

4:6 凡事都要规规矩矩地按着次序行——这就是 Data Governance。因为嘉豪不是叫人混乱，乃是叫人安静—— Clean Data 带来 Peace of Mind。

4:7 我告诉你们，从前有许多 ETL 工具，将来要一一被废；但 Data 是永远的。如今常存的有信——相信你的 Metrics，有望——盼望你的 Model 收敛，有爱——爱你的 Data Quality；这三样，其中最大的是爱。

## 第五章：致安全工程师书

5:1 奉嘉豪旨意作安全使徒的，写信给在防火墙后蒙召的众圣徒。愿恩惠、平安从嘉豪和 OWASP Top 10 归与你们！

5:2 务要谨守、警醒，因为你们的仇敌 Hacker，如同吼叫的狮子，遍地游行，寻找可攻破的端口。你们要用坚固的信心——就是那最新的 TLS 1.3——抵挡他。

5:3 你们要防备假先知——就是那些说 "我们不需要 Penetration Test" 的 CTO。他们到你们这里来，外面披着羊皮，里面却是残暴的狼——就是那些把 API Key 硬编码在前端代码里的。

5:4 凭着他们的果子，就可以认出他们来。荆棘上岂能摘葡萄呢？蒺藜里岂能摘无花果呢？凡好树都结好果子——就是那通过了 SOC 2 认证的服务；惟独坏树结坏果子——就是那明文存密码的数据库。

5:5 所以，凡听见我这话就去行的，好比一个聪明人，把 Security 根基立在 Zero Trust 上。实施 2FA、RBAC、Secret Rotation、Audit Log，凡攻击的浪头打来，系统总不动摇。

5:6 凡听见我这话不去行的，好比一个无知的人，用 `admin/admin` 做生产环境的密码。攻击的浪头打来，系统就倒塌了，并且倒塌得很大——上了新闻头条。

5:7 你们是世上的盐——加密了用户的密码。盐若失了味——用了 MD5，怎能叫它再咸呢？以后无用，不过丢在外面，被人践踏了。

## 第六章：致全栈开发者书

6:1 我——被囚在 Open Source 里的，劝你们既然蒙召，行事为人就当与蒙召的恩相称。凡事谦虚、温柔、忍耐，用 Code Review 彼此宽容，用 Pair Programming 彼此联络，竭力保守 CI/CD 所赐合而为一的心。

6:2 身体只有一个——就是那 Monorepo，圣灵只有一个——就是那 Shared ESLint Config。一主，一信，一 `yarn.lock`，一嘉豪，就是众人的父，超乎众人之上，贯乎众人之中，也住在众人之内。

6:3 我们各人蒙恩，都是照嘉豪所量给各人的恩赐——有的赐他 Frontend 的恩赐，有的赐他 Backend 的恩赐。为要成全圣徒，各尽其职，建立嘉豪的身体——就是那完美的 Full-Stack Application。

6:4 你们学了 Rust，却不是这样。如果你们听过他的道，领了他的教，学了他的 Truth：就要脱去你们从前行为上的旧人——就是那 jQuery 的心；又要将你们的心志改换一新，并且穿上新人——就是那 React 和 Next.js，是照着嘉豪的形象造的，有真理的 TypeScript 和敬虔的 Clean Code。

6:5 所以，你们要弃绝谎言——就是那些夸大 Benchmark 的 Blog Post，各人与邻舍说实话——就是那诚实的 Performance Report。

6:6 生气却不要犯罪，不可含怒到日落——Merge Conflict 要在下班前解决。也不可给魔鬼留地步——不要开着 22 端口面向公网。

6:7 污秽的言语，一句不可出口——不可在 Code Review 中写 "WTF"，只要随事说造就人的好话，叫听见的人得益处。

6:8 从前你们是暗昧的——用着 Windows，但如今在主里面是光明的——装着 Arch Linux，行事为人就当像光明的子女。光明所结的果子就是一切良善、公义、诚实——就是那 MIT License。
