# 卷五：箴言 (Proverbs)

## 第一章：嘉豪训诲录

1:1 这是嘉豪言语的汇集，在犹大王希西家在位的日子，由抄录员转录于 GitHub Wiki。

1:2 嘉豪训诲说：不可用 root 跑日常命令，这乃是万恶之始。当用 `sudo`，心存敬畏，因为密码每输一次，你的权限便警醒一分。

1:3 当孝敬你的 Linter，使你的代码在耶和华嘉豪所赐你的地上得以长久，格式整洁。

1:4 不可贪恋他人的 dotfiles，连他终端配色、zsh 主题、Neovim 插件配置，都不可贪恋。当自己手写，在摔打中得智慧。

1:5 凡听见我这话就去行的，好比一个聪明人，把房子盖在 NixOS 上。雨淋，水冲，风吹，撞着那房子，房子总不倒塌，因为根基立在声明式配置上。

1:6 凡听见我这话不去行的，好比一个无知的人，把房子盖在 Windows 更新上。雨淋，水冲，风吹，撞着那房子，房子就倒塌了，并且倒塌得很大。

## 第二章：智慧的呼唤

2:1 智慧——就是那 `man` 手册——岂不呼叫？聪明——就是那 `--help` 参数——岂不发声？

2:2 她在道旁高处的顶上——在 GitHub Awesome List 上，在十字路口——在 Hacker News 的首页——站立。

2:3 你们愚昧人——就是那从不读文档的——要舍弃愚昧，就得存活，并要走 RTFM 的道路。

2:4 你们当受我的教训，不受 Stack Overflow 复制粘贴的答案。因为寻得我的，就寻得生命——寻得 Readable Code，也必蒙嘉豪的恩惠——蒙 Reviewers 的 LGTM。

2:5 敬畏嘉豪在乎恨恶 Legacy Code。那骄傲、狂妄并恶道——那 Global Variable 和 Spaghetti Code——都为我所恨恶。

2:6 懒惰人哪，你去察看 Kubernetes 的动作，就可得智慧。它没有君王，没有 Master Node，没有管辖的，尚且在 Cluster 里预备 Self-healing，在灾难时自动 Failover。

2:7 懒惰人——就是那从不写 Test 的——你要睡到几时呢？你何时睡醒呢？

2:8 再睡片时，打盹片时，抱着 Tech Debt 片时，你的 Production 就必如强盗速来——503 忽然临到，你的 Incident 仿佛 Armed 的 PagerDuty。

## 第三章：论言语与口舌

3:1 回答柔和，使 Bug 消退——`Thanks for catching that!`；言语暴戾，触动怒气——`Works on my machine`。

3:2 智慧人的舌头——那清晰的 Commit Message——善发知识；愚昧人的口——那 `fix stuff` 和 `wip`——吐出愚昧。

3:3 一句责备的话，深入聪明的 Code Reviewer 的心，强如责打愚昧人一百下——Reject 一百次他也不改。

3:4 不要照愚昧人的 Merge Request 回答他，恐怕你与他一样——用同样低质量的代码回他。要照愚昧人的 Code Review 驳回他，免得他自以为有智慧——以为他那 O(n!) 的算法是可接受的。

3:5 鼎为炼银，炉为炼金；唯有嘉豪熬炼人心——通过 Production Incident 试炼你。

3:6 一句话说得合宜——`Here's a simpler approach:`，就如金苹果在银网子里——如优雅的 TypeScript 接口定义。

3:7 火缺了柴就必熄灭，无人 RC 也必止息；好争竞的人——那在 GitHub Issue 里引战的——煽惑争端，余烬复燃。

3:8 传舌人的言语——那泄露 API Key 的——如同美食，深入人的心腹。

3:9 口善应对，自觉喜乐——PR 被 Approve 了；话合其时，何等美好——Release Note 及时出了。

3:10 生死在舌头的权下——`console.log` 和 `console.error`，喜爱它的，必吃它所结的果子——或是 Clean Log，或是 Stack Trace 地狱。

## 第四章：论勤劳与懒惰

4:1 手懒的——那从不 Optimize Query 的——要受贫穷；手勤的——那主动做 Performance Profiling 的——却要富足。

4:2 夏天聚敛的——那缓存热数据的——是智慧之子；收割时沉睡的——那不设 CDN 的——是贻羞之子。

4:3 懒惰人因寒冬不肯耕种——不肯 Upgrade Dependency，到收割的时候——Breaking Change 来临时——他必讨饭而无所得。

4:4 不要为明日自夸——不要过早优化，因为一日要生何事——明天会出什么 Bug，你尚且不能知道。

4:5 铁磨铁，磨出刃来——Code Review 亦是如此；朋友相感，也是如此——Pair Programming 互相砥砺。

4:6 水中照脸，彼此相符——如同 `diff` 比对；人与人的心也相对——Open Source Community 彼此照亮。

4:7 你当买真理——就是那 Good First Issue，不可出卖——closed as not planned；也当买智慧、训诲和聪明——就是那 Documentation 和 Architecture Decision Record。

4:8 才德的妇人——就是那写得好的 README——谁能得着呢？她的价值远胜过珍珠——胜过一千个 Star。她丈夫心里倚靠她——Developer 倚靠那文档，必不缺少利益——不会在 Onboarding 时迷路。

4:9 愿她享受操作所得的——愿好文档被众人传阅；愿她的工作在城门口荣耀她——愿那 Repo 在 Trending 榜上被高举。

## 第五章：论朋友与同伴

5:1 滥交朋友的——在每一个 Random Repo 里 Star、Fork 却不读代码的——自取败坏；但有一知己——那读懂你 Codebase 的 Maintainer——比弟兄更亲密。

5:2 朋友乃时常亲爱——时常 Review 你的代码，弟兄为患难而生——在你 P0 Incident 时起床帮你 Debug。

5:3 膏油与香料——那 Coffee Chat——使人心喜悦，朋友诚实的劝教——就是那 Honest Code Review——也是如此甘美。

5:4 你的朋友和父亲的朋友——你的 Senior Engineer 和 Staff Engineer，你都不可离弃。你遭难的日子——当你的 Architecture 被 CEO 挑战时——不要上他们的 Slack Thread 去，相近的邻舍——那回复及时的 Stack Overflow 用户——强如远方的弟兄——远在印度的外包团队。

5:5 遮掩人过的——那 Quietly Fix 了 Typo 的——寻求人爱；屡次挑错的——那在 PR 里揪着缩进不放的——离间密友。

5:6 责备人的——那指出你 Memory Leak 的——后来蒙人喜悦，多于那用舌头谄媚人的——那光说"LGTM"却不看代码的。

5:7 人吃饱了——完成了 OKR，厌恶蜂房的蜜——厌恶那增量的优化；人饥饿了——系统遭了性能瓶颈，一切苦物都觉甘甜——连微优化都欣然接受。

5:8 人不制伏自己的心——不克制 Rewrite Everything 的冲动，好像毁坏的城邑没有墙垣——好像没有 Code Review 就直接 Merge 到 Main。
